Comparison to other React static site generators

Gatsby

Similar to Nucleate, Gatsby offers:

  • Creation of pages as React components, using ES2015+ via Babel
  • Creation of pages as Markdown documents
  • Declarative, directory-based routes
  • Instant client-side route transitions
  • Live reloading during development as you edit code

In addition, Gatsby also offers:

  • Hot module replacement via react-transform-hmr

    Note: I chose not to add HMR to Nucleate yet due to being unable to hot reload Markdown pages (due to issues described here by @gaearon ), and due to conflicts with Nucleate's server rendering. When react-hot-loader@3 is released and react-router#2182 is resolved, I'll reconsider HMR.

  • The gatsby new command to create a new site from a template

However, Gatsby some drawbacks:

  • Complex, opinionated configuration and folder structure: config.toml, html.js, /pages, _template.jsx, etc.
  • No server rending during development: this poor dev/prod parity can lead to different behaviour between the development site and the generated static site.
